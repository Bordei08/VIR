1. Introduction

1.1 Purpose

The current version number for this product is 1.0 (first version). The product is intended for helping users to view new videoclips or to add ratings to the videoclips they have seen and have a good opinion about.

1.2 Document Conventions

  • The document is based on the template IEEE System Requirements Specification Template .
  • Sequences of references are specified in italics.

1.3 Intended Audience and Reading Suggestions

The document is intended for users, developers and testers. We suggest reading the document for users in the current format, starting with the introduction. For developers and testers, we suggest reading the document with the "Overall Description".

1.4 Product Scope

The aim for the user is to accumulate knowledge about different fields of mathematics (algebra, geometry, trigonometry, analysis), by age categories (primary, middle school, high school). The problems presented in the application are divided into modules - light, medium and a difficult problem, which requires addiotional research on the solution technique. The user has the possibility to buy hints in the application, in order to test their basic knowledge.

2. Overall Description

2.1 Product Functions

  • Registration in the application
  • Application log in
  • Adding new ratings on your favortie movies
  • Adding new comments on your favorite movies
  • Adding new videos in your liked list

2.2 Users and Characteristics

We believe that the class of users who will use this application will be composed of any person alive since everybody might need a recommendation before trying something new.

2.3 Operating Environment

For optimal use of the application, version 1.0, we recommend using one of the latest version of your browser. Also, there is no need for top hardware specifications - due to the simplicity of the application, we believe that it can run on the vast majority of computers.

2.4 Design and Implementation Constraints

For the interface of the web application, we used only HTML5, CSS3 and JavaScript. Instead, to create the backend we will mainly use PHP.

2.5 User Documentation

In order for the web application to work in optimal parameters, it is necessary to use the one of the latest versions of mainly any browser.

2.6 Assumptions and Dependencies

The part of the database in which users's information is stored may affect the utility of the application (for example, if the database service requires a fee, depending on the complexity, changes will be made to the application).

3.2 Hardware Interfaces

Devices that can interact with the application include both computers (PCs, laptops, other similar types) and tablets and mobile devices. Because the application has a responsive interface, it can scale information to different screen sizes.

3.3 Software Interfaces

A first software component that establishes a connection with the application is the database. To access the database, we will use the PHP language, and to modify the data inside the database, we will use MySQL. Also, scripts written in JavaScript language are used to improve the user experience in the application.

3.4 Communications Interfaces

The application imposes the condition that the user has an internet connection, in order to be able to verify the answers given by him. Also, the internet connection is necessary regarding the contact with the application developers, in the contact window. Another protocol that is used within the application (on the server side) is the HTTP protocol.

4. System Features

4.1 System Feature 1: Save the liked videos

This feature has a high priority because it is the main feature of the application. In addition to the other features, the main purpose of the application is to save your favorites videos and see other opinions.

4.2 System Feature 2: See comments and ratings on a video

This feature has a medium priority because it might help the user decide if he wants maybe to see a video based on the reviews and comments.This is a good way to decide if you want to try something new or not and to add it to your list.

4.3 System Feature 3: Add new comments and ratings on a video

This feature has a medium priority because it only adds your opinion about the video in the comment section. This might be however important to other users.

5. Other Nonfunctional Requirements

5.1 Performance Requirements

Currently (version 1.0 of the application), the developers consider that there are no performance requirements, as long as the application runs in one of the latest versions of the browsers the user is using.

5.2 Safety Requirements

The results of testing the application in the current version (1.0) showed that there are no losses, damages or losses that may occur from the use of the application. However, if a violation of one of the above is observed, it is recommended to isolate and analyze the incident.

5.3 Security Requirements

The database used is MySQL, and each user's data is stored in the database. Each user's password is not stored properly, but is applied to a hash. The final result is stored in the database. Through certain functions that cannot be revealed publicly, each user logs in to his password.

5.4 Software Quality Attributes

The application has as secondary purpose reliability, but also portability on different devices (computer, tablet, phone). This work is done with the help of the responsive interface.

Other Requirements

1. Formalization of input data

In order for users' data to be stored correctly in the database, they must have the following formalities:

  • Email termination must have the suffix "@yahoo.com" or "@gmail.com"
  • To search a video you will have to use in the search bar a name as close as possible to the complete name of the video.
